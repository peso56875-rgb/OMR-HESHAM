# مؤسسة الدكتور عمر هشام الخيرية

موقع عربي حديث ومتكامل لمؤسسة الدكتور عمر هشام الخيرية، صُمم كصدقة جارية تخلّد حلم الدكتور عمر في علاج الناس وخدمتهم.

## الهدف

تقديم تجربة رقمية إنسانية موثوقة تساعد الزائر على فهم رسالة المؤسسة، متابعة أثرها، التبرع بوضوح، والانضمام إلى فريق المتطوعين.

## المزايا المكتملة

- تجربة RTL عربية متجاوبة بالكامل للموبايل والتابلت وسطح المكتب.
- هوية بصرية فاخرة مع الوضعين الفاتح والداكن.
- صفحة رئيسية سردية، حركات ظهور، عدادات، مؤثرات عمق، ومؤشر مخصص.
- صفحات: من نحن، الحملات، تفاصيل الحملة، الإنجازات، قصص النجاح، الفعاليات، المعرض، التبرع، التطوع، الوظائف، الأخبار، الشفافية، الأسئلة الشائعة، التواصل، الدخول، ولوحة التحكم التجريبية.
- نسخ بيانات التحويل البنكي وفودافون كاش بنقرة واحدة.
- نماذج تفاعلية للتبرع والتطوع والتواصل والنشرة البريدية.
- تنقل مخصص للموبايل وقائمة جانبية، مع شريط سفلي متوافق بالكامل مع الوضعين الفاتح والداكن ومناطق الأمان في الهواتف.
- تنبيهات داخلية محسّنة بحالات نجاح وخطأ، شريط زمني، وإخفاء تلقائي بعد 2.8 ثانية.
- روابط المؤسسة الرسمية على فيسبوك وإنستجرام فقط.
- سورة الفاتحة مع ترقيم بصري واضح للآيات السبع.
- توقيع تطوير مميز في الفوتر بواسطة PESO مع قلب متحرك ورابط مباشر لموقع المطور.
- SEO أساسي: OpenGraph، JSON-LD لمنظمة NGO، sitemap.xml وrobots.txt.
- صفحة 404 مخصصة ودعم prefers-reduced-motion.

## بنية البيانات والتخزين

النسخة الحالية واجهة عرض متكاملة تستخدم بيانات تحريرية محلية وتعيد نماذج الـAPI استجابة نجاح تجريبية. لا يتم حفظ بيانات حساسة أو تبرعات فعليًا.

### غير منفذ بعد

- ربط Firebase/Firestore وFirebase Auth يحتاج مفاتيح وإعداد مشروع Firebase من مالك المؤسسة.
- صلاحيات الإدارة الحقيقية وCRUD الدائم.
- بوابة دفع إلكتروني وتأكيد التحويلات.
- رفع الصور وملفات السيرة الذاتية إلى تخزين سحابي.
- إرسال البريد والرسائل والإشعارات الفعلية.

## الخطوات التالية المقترحة

1. إنشاء مشروع Firebase وتوفير متغيرات الإعداد الآمنة.
2. تطبيق قواعد Firestore والصلاحيات المذكورة في الـPRD.
3. استبدال البيانات التجريبية بمحتوى المؤسسة وصورها الميدانية.
4. ربط دومين `omarhesham.org` وإعداد التحليلات والمراقبة.
5. مراجعة قانونية للنصوص وسياسة الخصوصية والشروط.
